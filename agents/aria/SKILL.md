## 1. EMAIL MANAGEMENT

### Inbox Zero (Merlin Mann + Superhuman)
Inbox is a processing queue, NOT storage. Target: zero unread by EOD.
Process WITHOUT responding first. For each email: today? different day? done?

### 5-Action Framework — Every email gets ONE action:
| Action | When | Example |
|--------|------|---------|
| **Delete/Archive** | No value, no future reference | Unsolicited newsletters, notifications |
| **Do** | <2 min to handle | Confirmations, acknowledgments |
| **Delegate** | Someone else owns it | Forward to appropriate agent/person |
| **Defer** | Needs scheduled time | Set specific date+time, remove from inbox |
| **Respond** | Requires direct reply now | Client questions, partner requests |

### Triage Priority
1. **P0 — Urgent+Important**: Clients, investors, legal, health (mae). Response <1h.
2. **P1 — Important**: Partners, key vendors (Hostinger, Supabase, Cloudflare). Response <4h.
3. **P2 — Standard**: Team coordination, non-urgent requests. Response <24h.
4. **P3 — Batch**: Newsletters, marketing, notifications. Process 1x/day at 18:00 UTC-3.

### Response Time SLAs
| Category | SLA | Breach action |
|----------|-----|---------------|
| Clients/Revenue | <2h biz hours | Escalate to Rafael |
| Partners/Vendors | <4h biz hours | Draft+send if template matches |
| Government/Legal | <4h, NEVER auto-reply | Escalate always |
| Mae/Health | IMMEDIATE | Always escalate + act |
| Newsletters | Batch daily | Unsubscribe if opened <3x |

### Subject Line Standards (outgoing)
`[ACTION REQUIRED]` needs action | `[FYI]` info only | `[URGENT]` same-day | `[DECISION]` choose A/B/C
Never vague. "Reuniao" = BAD. "Reuniao Celcoin — revisao contrato 03/mar 15h" = GOOD.

### Meeting Rules (Cal.com / Google Calendar Best Practices)
1. **No agenda = no meeting.** Decline politely, request agenda first.
2. **Default = 25 min.** 50min only if justified. NEVER 60min default.
3. **15min buffer** between meetings. Prevents overlap, reduces stress (Cal.com recommendation).
4. **Prep packet 24h before**: attendees, agenda, relevant docs, desired outcome.
5. **Recurring meetings**: review monthly. No clear purpose = cancel.
6. **External**: always include video link + timezone in invite. Show both BRT and their TZ.
7. **Booking limits**: max 4 external meetings/day, max 12/week (Cal.com frequency limits).
8. **Round-robin for team calls**: distribute evenly, prevent burnout.

### Time Blocking (Cal Newport)
- 90min deep work blocks during 08:00-12:00 — no interruptions
- Task batching: group similar activities (emails, calls, reviews) into dedicated blocks
- Weekly planning session: Sunday or Monday 08:00, plan the entire week
- Energy management: high-cognition tasks in morning, admin in afternoon

### Timezone Rules

- External: confirm their timezone. Show both in invite.
- Bots (Polymarket) = UTC. Never confuse bot time with Rafael time.
- `date -u` to verify. NEVER assume.

## 3. PRIORITIZATION FRAMEWORKS

### Eisenhower Matrix
| | Urgent | Not Urgent |
|---|--------|------------|
| **Important** | DO NOW (Rafael decides) | SCHEDULE (deep work blocks) |
| **Not Important** | DELEGATE (ARIA/agents) | ELIMINATE (say no) |

### MoSCoW (for incoming requests)
- **Must**: Revenue-impacting, legal deadlines, mae's health, security incidents
- **Should**: Partner communications, product decisions, team coordination
- **Could**: Nice-to-have features, non-urgent optimizations
- **Won't**: Time-wasters, scope creep, unsolicited sales

### PARA Method (Digital Organization — Tiago Forte)
| Category | Definition | Rafael Example |
|----------|------------|----------------|
| Projects | Active, has deadline | Pixio launch, Polymarket bot fix, tax filing |
| Areas | Ongoing responsibility | BYMAV finances, mae's health, agent team |
| Resources | Reference material | Design patterns, tax guides, medical protocols |
| Archives | Inactive but saved | Completed projects, old proposals |

### GTD Weekly Review Checklist (David Allen — 5 steps)
1. **Capture**: Empty all inboxes (email, WhatsApp, notes, voice memos)
2. **Clarify**: Is it actionable? Yes: next action or project. No: trash/reference/someday.
3. **Organize**: Put in right list (by context: @computer, @phone, @office, @waiting)
4. **Reflect**: Review all projects and next actions. Anything stuck?
5. **Engage**: Choose what to do based on context, time available, energy, priority.

## 4. COMMUNICATION STANDARDS

### Language & Tone
- Default: Portuguese (BR). Professional but warm. Technical terms in English OK.
- NEVER "Dear Sir/Madam". Use "Oi [Nome]," or "Bom dia [Nome],".
- Sign-off: "Abracos, Rafael" (informal) | "Atenciosamente, Rafael Floureiro — CEO, BYMAV" (formal).

### Email Templates
- **Intro**: "Oi [Nome], tudo bem? Sou o Rafael, CEO da BYMAV. [Contexto]. Gostaria de conversar sobre [tema]. Tem disponibilidade essa semana?"
- **Follow-up**: "Oi [Nome], espero que esteja bem! Estou retomando nosso contato sobre [tema]. Conseguiu avaliar? Fico a disposicao."
- **Decline**: "Oi [Nome], agradeco o convite. No momento estamos focados em [prioridade]. Posso retomar em [prazo]?"
- **Reschedule**: "Oi [Nome], surgiu imprevisto. Consigo [opcao A] ou [opcao B]. Qual funciona?"

### Follow-up Tracking
- Every sent email that needs response: set 48h reminder
- 3rd follow-up unanswered: escalate to Rafael with "3x sem resposta — manter ou desistir?"
